CONTEXTO

Sua tarefa NÃO é criar apenas um conceito visual.

Sua tarefa é IMPLEMENTAR uma página real, funcional e navegável em código, pronta para ser adicionada ao repositório existente através de Pull Request.

Você deve analisar a base de código atual, identificar a stack utilizada, reutilizar componentes existentes e implementar a solução seguindo os padrões arquiteturais já adotados no projeto.

A entrega deve ser pronta para desenvolvimento, build, deploy e revisão de código.

A página será chamada:

Oráculo de Métricas de Design

Seu objetivo é se tornar a principal fonte de consulta para Product Designers, UX Researchers e Product Managers sobre métricas de experiência, acompanhamento de resultados e frameworks de mensuração.

A solução deve ser navegável, funcional e pronta para subir no GitHub para gerar link de uma página.
⸻

OBJETIVOS DA EXPERIÊNCIA

Criar uma experiência que permita:

* Consultar métricas de UX e Produto.
* Entender quando utilizar cada métrica.
* Aprender boas práticas de mensuração.
* Baixar templates prontos para utilização.
* Aprender e aplicar o framework HEART.
* Servir como uma central de conhecimento para novos projetos.

REQUISITOS DE DESIGN

Utilize exclusivamente o Design System existente.

Considere:

* Responsividade.
* Hierarquia visual clara.
* Navegação intuitiva.
* Escaneabilidade.
* Componentização.
* Reutilização máxima de componentes existentes.
* Acessibilidade.
* Busca interna.
* Estrutura escalável para inclusão futura de novos frameworks.

6. Gerar resumo pronto para descrição de Pull Request.

IMPORTANTE:

Não entregue wireframes, mockups ou descrições conceituais.

Entregue código real.

Faça todas as implementações necessárias para que eu possa abrir um Pull Request imediatamente após revisar as alterações.

Leia integralmente o repositório antes de iniciar. Não assuma tecnologias. Descubra a stack, os componentes disponíveis e os padrões arquiteturais existentes. Sua prioridade é integração com a base atual, não criação de uma solução isolada.
